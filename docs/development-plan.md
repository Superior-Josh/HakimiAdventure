## Sprint 0 — 项目骨架 & 移动相机

> **目标**：搭建 Godot 项目，玩家在 3D 场景中自由移动。是一切代码的起点。

| # | 任务 | 依赖 | 工时 |
|---|------|------|------|

| 0.2 | Git 仓库 + `.gitignore` + 初始提交 | 0.1 | 0.5h |
| 0.3 | **PlayerController**：WASD 移动 + 鼠标视角 + 重力/碰撞 | 0.1 | 3h |
| 0.4 | **CameraController**：FOV 可调 + head bob + 受击抖动接口 | 0.3 | 2h |
| 0.5 | **Input Map**：全部 action 命名定义（move/attack/interact/lock_on/use_item 等） | 0.3 | 0.5h |

| 0.7 | **AudioManager 骨架**：AudioBus 三轨（BGM/SFX/Voice）创建 | 0.1 | 1h |

| 0.9 | **测试场景**：greybox 房间 + 地面 + 光源 + 墙壁碰撞 | 0.1 | 1.5h |

## Sprint 1 — 战斗核心闭环

> **目标**：即时体力制战斗 + 一个敌人 + 锁定。打完就能"砍怪"。

| # | 任务 | 依赖 | 工时 |
|---|------|------|------|
| 1.1 | **CombatSystem**：状态机（Idle → Attacking → Hit → Dead），前摇/后摇/硬直逻辑 | S0 | 3h |
| 1.2 | **StaminaSystem**：体力上限/消耗/恢复速度，可配置 Resource | 1.1 | 1.5h |
| 1.3 | **WeaponController**：攻击射线/碰撞盒判定 + 连击窗口 | 1.1 | 3h |
| 1.4 | **LockOnSystem**：锁定最近敌人 + 切换目标 + 相机跟随转向 | 1.1, 0.4 | 2.5h |
| 1.5 | **敌人预制体**：占位模型 + AnimationPlayer + 碰撞体 + 可配置数据 Resource | 1.1 | 2h |
| 1.6 | **EnemyAI**：状态机（巡逻→发现→追踪→攻击→硬直→死亡） | 1.5 | 4h |
| 1.7 | **DamageSystem**：伤害公式 + 玩家/敌人 HP 管理 + 受击接口 | 1.3, 1.6 | 2h |
| 1.8 | **HUD**：HP 条 + 体力条 + 锁定指示器 | 1.2, 1.7 | 2h |

### 依赖图

```
S0 → 1.1 (CombatSystem) ─┬→ 1.2 (StaminaSystem)
                          ├→ 1.3 (WeaponController)
                          ├→ 1.4 (LockOnSystem)
                          └→ 1.5 (Enemy预制体) → 1.6 (EnemyAI)
                                                    │
1.3 + 1.6 → 1.7 (DamageSystem) → 1.8 (HUD) ← 1.2
```

### 🏁 交付物

> **战斗原型**：玩家靠近敌人 → 锁定 → 攻击（消耗体力，有前摇命中判定）→ 敌人受伤/反击 → 任一方 HP 归零。HUD 显示 HP/体力/锁定指示器。

**预估：20 小时**
